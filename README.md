# TimeSaver 
TimeSaver is a mobile time-tracking application designed to help users record the time they spend working on projects, manage productivity and monitor progress toward daily goals. It was built as a group project in Android Studio using Kotlin. The goal of this application was to implement a functional time-tracking system while demonstrating strong understanding of backend development, database integration, UI design and general software engineering practices.




# Usage 
Sign Up/Login: Create a new account or log in with an existing account.
Create Categories: Navigate to the categories section and create categories that fit your tasks.
Add Timesheet Entry: Go to the timesheet entry section to log your work hours, descriptions, and optional photos.
Set Daily Goals: Set your minimum and maximum daily work hour goals in the settings section.
View Reports: Check the reports section to see your timesheet entries, total hours per category, and daily work hours graph.
Track Performance: Monitor your performance against your daily goals over the past month.

# User Authentication
Login: Users can log in to the app using a username and password.
Signup: New users can create an account with a username and password.

# Timesheet Management
Categories: Users can create categories to organize their timesheet entries.
Timesheet Entry: Users can create timesheet entries with the following details:
Date
Start and end times
Description
Category
Optional photo attachment
Daily Goals: Users can set minimum and maximum daily goals for hours worked.

# Reporting and Visualization
Timesheet List: Users can view a list of all timesheet entries for a user-selectable period, including access to any stored photos.
Category Hours: Users can view the total number of hours spent on each category during a user-selectable period.
Daily Hours Graph: A graph displays the total hours worked each day over a user-selectable period, along with minimum and maximum goals.
Performance Visualization: The app visually displays how well the user is staying within their daily hour goals over the past month.

# Data Storage
Online Database: All data is stored securely in an online database to ensure accessibility and reliability.
Gamification Features
Achievements: Users can earn achievements for consistent time tracking, meeting goals, and other milestones.
Leaderboard: A leaderboard shows how users rank in terms of hours tracked and goals met, fostering a sense of competition and motivation.



# Own Additional App Features 

## Feature 1: Notifications 
The user is able to view notifications on the application. When the user is successfully logged in and presented with the main menu, there is a menu option called 'Notifications'. If the user clicks on 'Notifications', they will be brought to a page where they are able to view a list of notifications relating to their timesheets. The notifications will signify to the user when they need to start working on a timesheet, when they can stop working on a timesheet, what their goals are for the day, etc. The notifications contain the date and time of when they are sent out to the user. 

## Feature 2: Achievements
The user is able to view their achievements on the application. When the user is successfully logged in and presented with the main menu, there is a menu option called 'View'. If the user clicks on 'View', they will be directed to another menu with options. One of the options on this sub-menu is 'Achievements'. If the user clicks on 'Achievements', they will be directed to a page with various achievements that the user can accomplish through using the app. A user has accomplished an achievement if the check box next to a specific achievement is ticked. The system will automatically tick or untick a box when a user reaches an achievement or the achievement resets for a new day, month or year. There are daily achievements, monthly achievements, yearly achievements and overall achievements that the user can accomplish. 

## Feature 3: Contact Us
The user is able to send a message to the people behind the application to query something or provide feedback. When the user is successfully logged in and presented with the main menu, there is a menu option called 'Contact Us'. If the user clicks on 'Contact Us', they will be brought to a page where they are able to input an email address and a message about their query or feedback.
